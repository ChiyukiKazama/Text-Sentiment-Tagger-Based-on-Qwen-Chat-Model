# 设计与思考

## 数据集与任务描述

本项目选择了公开的**ChnSentiCorp.csv**作为数据集，其包含数千条不同情感的中文评论；

输入特征为纯文本，根据文本的**情感倾向**为其生成“正面”、“负面”和“未知”标签。

## 模型与方法

使用通义千问的**Qwen-7B-Chat-Int4**模型，该模型是一个经过对话场景专门优化和4-bit量化的开源LLM；

本项目中部署该模型进行本地推理，使用以下提示词约束模型输出以便提取情感标签：

```
"你是一个情感分析助手。请判断以下评论的情感倾向。\n"
"要求：只输出两个字，只能是“正面”、“负面”之一，不要输出标点符号和其他内容。\n"
"现在请分析：\n"
```

另外，通过设置以下超参数使模型采取**贪婪解码**策略，只选择概率最高的token作为下一token输出：

```
do_sample=False
```

保证模型“思维收敛”、输出稳定。

原始评论文本和打标后的评论文本以csv格式分别保存在’example'下的'input'和’output'文件夹中。

## 结果分析

通过运行test.py验证打标准确度，总体准确度在90%以上，并存在以下特征：

+ 能较好地理解上下文语境，对于包含多处转折的句子能做出较为合理的判断
+ 对于反语、阴阳怪气的理解仍有较大进步空间

综上，本项目实现了对文本的自动生成标签功能。

## 未来改进方向

+ 后处理模块：修改提示词使模型输出标签的同时输出对应置信度，并增加一个模块用于过滤低置信度标签以直接去除或重审；
+ 扩展标签体系：本项目中初步将情感分为正负两种倾向，未来可以扩展为细粒度更高的情感维度；
+ 升级模型：尝试使用同等算力需求的更新更强的模型（如Qwen3.5系列）以进一步提升准确度和鲁棒性...
